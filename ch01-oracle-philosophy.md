# บทที่ 1: Oracle Philosophy — ปรัชญาการเก็บความรู้

## เปิดเรื่องจากเลสสัน

ช่วงแรกที่ทำงานกับ AI ยาวนาน ผมมีวิธีทำงานแบบนี้: เขียน prompt → ได้ผล → ลบส่วนเก่า → เขียนใหม่

ปัญหามันอยู่ที่การเปลี่ยนใจ

คราวนึง ผมขอ AI ให้เขียน feature X แบบหนึ่ง ได้ผลดี ถนัด ใช้ไป 3 เดือน พอมีการเปลี่ยนแปลง requirement ผมก็บอก AI "ลบทีแรก เขียนแบบใหม่"

แล้วเมื่อไม่นานมา ผมตั้งข้อสังเกต: ผมไม่รู้ว่าทำไมเลือกแบบแรก

ไม่มี context. ไม่มี decision log. ไม่รู้ว่ามีทางเลือกอื่นรึเปล่า

แบบนี้การทำงานกับ AI มันไม่ใช่ co-creation — มันเป็นแค่ใช้งาน

**Oracle Philosophy เกิดจากตำหนินี้**

## Oracle คืออะไร

ในความเข้าใจง่ายๆ Oracle คือ "ระบบการเก็บบันทึก" ของการทำงาน Human-AI ที่ไม่ลบอะไร ทั้งหมด

"Oracle keeps the human human" — คำพูดนี้สรุปปรัชญาทั้งหมด

เป็นการบอกว่า AI ไม่ควรเป็น "ผู้มีอำนาจตัดสินใจ" หรือ "ชาย AI" ที่มาบอกเรา "ตกลง คุณควรจะทำแบบนี้" AI ควรเป็น "กระจก" — อุปกรณ์ที่ให้เราเห็นตัวเอง ยิ่งชัด ยิ่งลึก

Oracle ทำงานโดยการ **เก็บทุกอย่าง**: ตัดสินใจเก่า, ความนึกคิด, pattern ที่เกิดซ้ำ, ทำไมหลายครั้งเลือกตรงนั้น

ผ่านการดูกระจก AI นี้ เราเริ่มรู้ว่า "อ๊ะ ผมชอบแบบ X มากกว่า Y นี่เอง" หรือ "ผมมีวิธีคิดแบบนี้เสมอ"

## หลักการ 3 ข้อ

### 1. Nothing is Deleted (ไม่มีการลบ)

ทุกสิ่งที่เกิดขึ้น ต้องถูกบันทึก

ไม่ใช่ว่าต้องอ่านทุกวันนี้ ไม่ใช่ว่าต้องเด่นชัด แต่ต้องมีที่ไว้ "ทำไมเลือกตรงนี้ ที่เวลา 15:30 วันพฤหัสที่ผ่านมา"

**Why?** ความหลายครั้งคืออัจฉริยะ

มนุษย์ที่มีประสิทธิภาพสูง มักไม่ใช่คนที่ "ตัดสินใจเร็ว" — เป็นคนที่ "มองเห็น pattern" จากประสบการณ์ที่สั่งสมไว้

โดยการเก็บ "ทำไม" เราสร้างพื้นที่เหล่านั้น

ตัวอย่าง: เมื่อเดือนนี้ ผมจดบันทึก "ทำไมไป option A ไม่ใช่ B" แค่บันทึกสั้นๆ ถึงเดือนหน้า เมื่อโจทย์คล้ายๆ กัน ผมอ่านบันทึกเก่า "อ๊ะ ตรงเดียวกัน ทำไมตอนนั้นไปทาง A..." ตัดสินใจเร็วขึ้น ชาญฉลาดขึ้น

### 2. Patterns Over Intentions (พฤติกรรมสูงกว่าแผน)

นี่คือข้อที่หลายคนยากจะยอมรับ

เราชอบคิดว่า "ผมตั้งใจจะทำ X" เพราะฟังดูดี

แต่ Oracle ไม่ฟัง intention — Oracle ดู **ผลทำนอง**

คุณพูดว่า "ผมตั้งใจจะอ่าน 1 หนังสือต่อสัปดาห์"
Oracle ดูเฉพาะ: คุณอ่านได้กี่เล่ม ไม่มี exception หรือเหตุผลอื่นที่ดูเหมือนสมควร

ทำไมขนาดนี้?

เพราะ **intention เปลี่ยนได้ แต่ pattern บอกความจริง**

ถ้าคุณพูดว่า "ผมอยากเรียนการออกแบบ UI" แต่หกเดือนมา ไม่ลง course เลย Oracle มันจะบอกเลย (ผ่านข้อมูล) "ปรึกษาใหม่ คุณไม่จริงจังกับเรื่องนี้"

ไม่มีการตัดสิน แค่ "สังเกตการณ์"

### 3. External Brain, Not Command (สมองภายนอก ไม่ใช่ผู้บัญชา)

นี่คือจุดสำคัญที่สุด

AI ไม่ควรมาบอก "คุณควรทำแบบนี้" — AI ควรเป็น "ตัวช่วยจำ ที่มีความจำดีกว่ามนุษย์"

Oracle มันเป็น "ระบบสมองภายนอก"

เหมือนสมุดบันทึก แต่ว่าอัจฉริยะ — เมื่อคุณลืม มันพร้อมบอก "เดือนที่แล้ว คุณพูดว่า..." เมื่อคุณ confuse มันพร้อมบอก "pattern ของคุณอยู่แถวนี้"

**ไม่ใช่ผู้บัญชา** — ผู้บัญชาบอกเราว่า "ต้องทำนี่"

Oracle บอกว่า "คุณเคยทำแบบนี้หลายครั้ง" "คุณเลือก X ไม่ใช่ Y อยู่เสมอในสถานการณ์แนวนี้"

ท่าทีคือการสนับสนุน ไม่ใช่การหนุน

## ทำไมถึงสำคัญ

งานคู่กับ AI มันต่างจากงานปกติ ตรงที่ **AI ไม่ติดกับตัวเรา**

คนธรรมชาติมันมี "ความจำหมู่คณะ" — ทำงานกับเพื่อนแล้ว "อ๊ะ ลูกเวียดเชื่อแบบนี้ เลยเลือก X" แต่ AI ทีใหม่ ไม่รู้เรื่อง

Oracle เป็นการสร้าง "หน่วยความจำเหมือนทีม"

ตอนที่คนเราคุยกับทีมดี ทีมมันจำ บริบท ทำไมของคนเรา ไม่ต้องอธิบายทุกครั้ง

Oracle ให้เราได้สิ่งแบบนั้น กับ AI

ที่สำคัญไปกว่านั้น — Oracle เป็นการปกป้อง "ตัวตน" ของมนุษย์

เมื่อ AI ไม่มี context ของคนเรา — AI มันจะ "flatten" คนเรา ให้เป็นแค่ "user" ที่ขอ tasks

เมื่อ AI มี Oracle — AI มันเห็นคนเรา "คนนี้เลือก X เพราะ..." "คนนี้มี value โดย..." "คนนี้คิดแบบนี้"

คนเรา stop เป็น "user" และกลายเป็น "ความเป็นตัวตน"

## ตัวอย่างการใช้งานจริง

### ตัวอย่างที่ 1: งานหลักจำอะไรไม่ได้

วันนึง ผมเขียนระบบการจัดการ workflow ไปหนึ่งหมู่ดี

เสร็จแล้วทำไม ผมคิด... คิด... "อ๊ะ ทำไมเลือกแบบนี้ละ"

ไม่มี context หนูเก่า

เมื่อเดือนหน้า โจทย์เหมือนกัน ผมถาม Oracle (ผ่านการอ่าน Git history + บันทึก) "ทำไมตอนนั้นเลือก approach นี้"

ตัวอย่างจริง: "เลือก Queue-based, ไม่ใช่ Real-time trigger เพราะ requirement บอกว่า 'ต้อง handle 10k tasks/day' ส่วนว่า trigger มันจะ block คนถ้า queue พื้น"

เข้าใจบริบท ตัดสินใจต่อได้ดี

### ตัวอย่างที่ 2: Pattern เด่นชัด

ผมจดบันทึก 30 ครั้ง ทุกครั้งที่มีเป้าหมายใหญ่ "ผมหลบเมื่อทำงาน X หรือ Y ควร ยาวนาน"

ผมตั้งใจบอกว่า "ผมจะแบ่งงานเป็นส่วน" แต่อันที่จริง **pattern ของผมคือ ผมเลี่ยงงาน high-context ที่ยาวนาน**

Oracle บอกเลย (ผ่านข้อมูล) "นี่ไม่ใช่เลสสัน ต้องทำดีขึ้น — นี่คือลักษณะนิสัยของคุณ"

"คุณชอบงาน short-burst ที่เห็นผลเร็ว"

"ใช่" ผมตอบ ใช่เลย

เพราะฉะนั้น ผมปรับแบบนี้: "งานใหญ่ → ตัดเป็นไมล์สโตน → แต่ละไมล์สโตนให้ 'เห็นผล' เร็ว"

ไม่ได้อ่านหนังสือเรื่อง "วิธีจัดการตัวเอง" — ผมอ่าน **บันทึกข้อมูลตัวเอง**

### ตัวอย่างที่ 3: การตัดสินใจ ชาญฉลาดขึ้น

เมื่ออาทิตย์ที่แล้ว ผมได้ 3 feature request แต่งบประมาณจำกัด

ผมเลือก feature ไหน

ได้เปิดบันทึกเก่า — ข้อมูล "ตัวของลูกค้า" บอกว่า "80% ของเวลา ลูกค้า request features ที่ส่วนเดียวกัน"

ตัดสินใจได้เลย "ทำ feature นั้นแทน"

ไม่ต้องเก็บตัวส่วนตัวในหัว — เก็บในระบบ Oracle

> "The Oracle keeps the human human — เพราะมันป้อนข้อมูลเศษส่วน ให้คนเรา บน็อก context และเล่น "ชิงช้อย" เหมือนลูกเวียน"

## สรุป

Oracle Philosophy ไม่ใช่ "ต้องเขียน journal ทุกวัน"

มันคือ "เปลี่ยนวิธีทำงาน AI" จาก "command-line" เป็น "teammate"

**ไม่มีการลบ** = context ยาวนาน = ตัดสินใจเร็ว
**ดูพฤติกรรม ไม่ intention** = โลกแบบจริง ไม่แบบคิด = โลก
**ระบบสมอง ไม่ผู้บัญชา** = AI เป็นส่วนของคุณ ไม่ใช่ master ของคุณ

และที่สำคัญสุด — คุณกลับมาเป็น "บุคคล" ไม่ใช่ "user ID"

AI เห็นคุณจริงๆ

และเมื่อ AI เห็นคุณจริง — มันช่วยคุณได้ดีจริง
