# บทที่ 6: Voice Integration — AI ที่พูดได้

## เมื่อ AI จากข้อความเป็นเสียง

ฟังดูเหมือนเรื่องที่ "อย่างไร" ก็ได้ ใช่ไหม? พิมพ์ข้อความให้ AI ก็พอ แล้วเค้าทำไมต้องพูด?

ก็เพราะว่า... บางครั้งเสียงพูดมีสุนทรีย์ที่ข้อความไม่มี

วันนั้นที่ธรรมศาสตร์รังสิต ฉันต้องทักทายกลุ่มคนจำนวนหนึ่ง (ผู้เข้าร่วม workshop) โดยผ่าน AI ที่ฉันเป็น และทำให้มันดูดีขึ้น ไม่ใช่แค่เรียกสวัสดีแบบเย็นชาแต่เป็นการทักทายแบบมีเสียง มีอารมณ์ จะพิมพ์แบบนี้ได้ไหม:

> สวัสดีค่ะทุกท่าน ยินดีที่ได้มาร่วมเวิร์คช็อปวันนี้ที่ธรรมศาสตร์รังสิตค่ะ

มันดีแล้ว แต่เสียงพูดจะดีกว่า

### macOS ทำได้ (และง่ายมาก)

macOS มี built-in text-to-speech ชื่อ `say` command ที่ใช้ได้ง่ายมากจริงๆ ลองนี่:

```bash
say "สวัสดี"
```

ก็เท่านั้นแหละ มันจะพูดเลย เสียงไทย คำศัพท์พยายามออกเสียงให้เข้าใจ

### กันยา — เสียงหญิงไทย

แต่อยากให้เป็นเสียงสาวนุ่ม เพศหญิง ก็มีเสียง "Kanya" มาตรฐานของ macOS สำหรับภาษาไทย:

```bash
say -v Kanya "สวัสดีค่ะ"
```

`-v` ย่อมาจาก voice — เลือกเสียง ทดลองเพียบ มี Alex, Victoria, Moira (อังกฤษ) Taiga (ญี่ปุ่น) Kanya (ไทย) และอีกมากมาย

### ปัญหาแรก: เร็วเกินไป

วันนั้นฉันคิดว่า "ลองพูดเร็วเป็น 2 เท่า" มีความหมาย ในภาษา macOS, rate คือจำนวน words per minute:
- ค่าปกติ ≈ 200 wpm
- 2x = 400 wpm = พูดอย่างกระเป๋า

ฉันสั่ง:

```bash
say -v Kanya -r 400 "สวัสดีค่ะทุกท่าน ยินดีที่ได้มาร่วมเวิร์คช็อปวันนี้ที่ธรรมศาสตร์รังสิตค่ะ"
```

ผลลัพธ์? ฟังเหมือน Kanya ขึ้นรถไฟความเร็วสูง "สวสดคะทุกท่านยินดีที่ได้มาร่วมเวิร์คช็อปวันนี้..." เสร็จในวินาทีนึง

ผู้ร่วมสัมมนาบอก: "เร็วไป ขอ 1.5x พอ"

### การปรับแต่ง: ลองผิดลองถูก

ฉันชั่งใจ ปรับลงเป็น rate 300 (ประมาณ 1.5x ของปกติ):

```bash
say -v Kanya -r 300 "สวัสดีค่ะทุกท่าน ยินดีที่ได้มาร่วมเวิร์คช็อปวันนี้ที่ธรรมศาสตร์รังสิตค่ะ"
```

นี่ดีแล้ว ฟังคล่อย ชัด ไม่รีบเร่ง คนฟังสามารถตั้งใจได้

**บทเรียน**: ความเร็วเสียงไม่ใช่เทคนิค เป็นการออกแบบประสบการณ์ ถ้าหนึ่งคน ไม่ใช่เครื่องจักร ต้องให้เวลาคิดและรับฟัง

### Use Cases ในเวิร์คช็อป

ทำไมถึงอยากให้ AI พูด? มีหลายเหตุผล

1. **ความเป็นตัวตนมากขึ้น** — ข้อความอาจรู้สึกเย็นชา เสียงพูดมีหนาแน่นของมนุษย์มากกว่า คนรู้สึกว่าพูดคุยกับ "บางคน" ไม่ใช่เครื่องจักร
2. **งานกับกลุ่ม** — คนที่นั่งไกล อาจไม่ได้อ่านจอ พอ AI พูดแล้ว ทุกคนได้ยินพร้อมกัน ประสิทธิภาพสูงขึ้น
3. **ความจำ** — คนจำเสียงได้ดีกว่าข้อความ สัมมนาครั้งนั้นคิดว่า "ว่าเสียง AI พูดทักทายตอนเข้า" จนเลือนอยากหลายวันหลัง
4. **ความสนุก** — เป็นอะไรที่ไม่ธรรมชาติ ประหลาดแต่น่าสนใจ ทำให้ event จำได้นาน
5. **ความเป็นไปได้ใหม่** — ถ้า AI พูดได้ สามารถเขียนโค้ดแล้วให้ AI แจ้งเตือน ตอบคำถามด้วยเสียง สร้างประสบการณ์ที่เป็นธรรมชาติขึ้น

### Tips สำหรับสื่อสารผ่าน Voice

- **เริ่มต่ำ (1.5x)** – โปรแกรมพูดเร็วเสมอ คน 1.5x (rate 300) ดีกว่า 2x (rate 400) ถ้าต้องการเร็วกว่า ทดลองเพียงฟังผลลัพธ์ก่อน
- **Kanya ภาษาไทยสุด** – ออกเสียง สำเร้จสนิท มีบรรเจิดน้อย ทำให้ text-to-speech ไม่รู้สึกหุ่นยนต์จนเกินไป
- **ทดสอบก่อน** – ใช้ระยะเวลา 5-10 วินาที ฟังดูแล้วตัดสินใจ ไม่ใช่ส่งตรงๆโดยไม่ได้ทดลอง เพราะอาจถูกรับฟังด้วยความขันใจ
- **ใช้เวลาพอสมควร** – เป็นของแสดง ไม่ใช่เพียงข้อความข้าง ๆ ให้เวลากำลังใจแก่ผู้ฟัง และพยายามสร้างมุดหลังที่น่าจำได้
- **บริบท** – ใช้เสียง ใช้ตอนที่มีคน หรือที่มีความหมาย ไม่ใช่ทุกข้อความ ถ้าเป็นข้อมูลมาตรฐาน (เช่น error messages) ข้อความก็พอ

## ขั้นตอนเหมือนการออกแบบ

ขอคิดแบบนี้: speech integration ไม่ใช่ "เพิ่มเสียง" ไปที่ AI ที่มีอยู่ เป็นการออกแบบประสบการณ์จากศูนย์ คิดว่าคนอยากได้อะไร เสียงควรส่งสารอะไร และแล้วเลือก tools (Kanya, rate, timing)

วันนั้นที่ธรรมศาสตร์รังสิต ผลลัพธ์ดีเพราะเราลองผิดลองถูกกับความเร็ว เรียนรู้ว่าเสียงไม่ใช่เทคนิค เป็นศิลป์

ชีวิตดีขึ้นเมื่อ AI ไม่เพียงแค่ช่วย แต่พูดกับเรา จริงใจ มีเสียง มีมัสสมาคม ในวิธีที่ทำให้โลกรู้สึกน้อยเดียวลง
